## 2. 两阶段训练策略

### 核心原则

- **阶段一**：用 Human 小数据集快速验证模型架构能否跑通、loss 是否下降、val 指标是否合理
- **阶段二**：用 MERCaptionPlus 大数据集正式训练，Human 作为最终 test 集计算 F1 分数

### 为什么要分两阶段

| | 阶段一 (Human) | 阶段二 (MERCaptionPlus) |
|---|---|---|
| 目的 | 验证模型可行性 | 正式训练、提升性能 |
| 训练集 | ~1,226 条 | 最多 ~28,195 条 |
| 验证集 | ~306 条 | ~3,133 条 |
| 测试集 | 无 | Human 全量 1,532 条 |
| 预估耗时 | 30-50 分钟 | 数小时（取决于 ratio） |
| 判断标准 | loss 下降、val 指标合理 | test_wheel_f1 是否提升 |

## 3. 阶段一：快速验证（Human 数据集）

### 3.1 数据划分

```
Human 总样本: 1,532
├── 训练集 (80%):   1,226 条  →  ratio=0.3 后实际使用 ~368 条
├── 验证集 (20%):     306 条
└── 无测试集
```

- `enable_val_split: true` + `val_ratio: 0.2` → 自动 8:2 划分
- `ratio: 0.3` → 只用 30% 训练数据，加速验证
- 验证集不受 ratio 影响，始终使用全部 306 条

### 3.2 配置文件

`train_configs/human_cmef_tpa.yaml`

### 3.3 运行命令

```bash
CUDA_VISIBLE_DEVICES=0 python train.py \
  --cfg-path train_configs/human_cmef_tpa.yaml \
  --options \
  datasets.human.ratio=0.3 \
  run.iters_per_epoch=200 \
  run.max_epoch=10 \
  run.warmup_steps=100
```

### 3.4 验证通过标准

- [ ] 模型正常初始化，无报错
- [ ] train loss 持续下降（如 1.7 → 0.5 以下）
- [ ] val loss 不持续上升（允许波动）
- [ ] val_micro_f1 > 0.1
- [ ] val_wheel_f1 > 0.4
- [ ] 生成式输出格式正确（如 "The character's emotional state is ..."）

## 4. 阶段二：正式训练（MERCaptionPlus + Human test）

### 4.1 数据划分

```
MERCaptionPlus 总样本: 31,327
├── 训练集 (90%):   28,195 条  →  ratio 控制实际用量（见下表）
│   └── train_monitor (2%): ~56 条
└── 验证集 (10%):    3,132 条

Human 总样本: 1,532
└── 测试集:        1,532 条  (split_role: test，不参与训练和调参)
```

- MERCaptionPlus 的 `split_role: train` → 负责训练和验证
- Human 的 `split_role: test` → 全部作为最终测试集
- 训练结束后自动在 Human test 上跑生成式评测，输出 F1 分数

### 4.2 数据量选择

通过 `--options datasets.mercaptionplus.ratio=X` 控制训练数据量：

| ratio | 实际训练样本 | 建议配置 | 预估耗时 (单卡4090D) |
|-------|------------|---------|-------------------|
| 0.1 | ~2,820 | iters=1000, epoch=10 | ~2 小时 |
| 0.3 | ~8,459 | iters=1000, epoch=10 | ~3 小时 |
| 0.5 | ~14,098 | iters=2000, epoch=20 | ~6 小时 |
| 1.0 | ~28,195 | iters=5000, epoch=20 | ~12 小时 |

### 4.3 配置文件

`train_configs/mercaptionplus_cmef_tpa.yaml`（默认 ratio=0.3）

### 4.4 运行命令

```bash
# 30% 数据（默认配置，直接跑）
CUDA_VISIBLE_DEVICES=0 python train.py \
  --cfg-path train_configs/mercaptionplus_cmef_tpa.yaml

# 50% 数据
CUDA_VISIBLE_DEVICES=0 python train.py \
  --cfg-path train_configs/mercaptionplus_cmef_tpa.yaml \
  --options \
  datasets.mercaptionplus.ratio=0.5 \
  run.iters_per_epoch=2000 \
  run.max_epoch=20 \
  run.warmup_steps=1000

# 全量数据
CUDA_VISIBLE_DEVICES=0 python train.py \
  --cfg-path train_configs/mercaptionplus_cmef_tpa.yaml \
  --options \
  datasets.mercaptionplus.ratio=1.0 \
  run.iters_per_epoch=5000 \
  run.max_epoch=20 \
  run.warmup_steps=2000
```

### 4.5 输出指标

阶段二训练结束后自动输出以下指标：

| 指标 | 含义 | 关注度 |
|------|------|--------|
| test_wheel_f1 | 情感层级匹配 F1（竞赛核心指标） | **最重要** |
| test_micro_f1 | 精确标签匹配 F1 | 重要 |
| test_exact_match_acc | 完全匹配准确率 | 参考 |
| test_loss | 测试集 loss | 参考 |

### 4.6 结果文件位置

```
output/mercaptionplus_cmef_tpa/<实验时间戳>/
├── log.txt                    # 训练日志
├── checkpoint_best.pth        # 验证集最优 checkpoint
├── curves/
│   ├── metrics_history.json   # 结构化指标历史
│   └── training_curves.png    # 训练曲线图
└── experiment_summary.md      # 实验总结（如有）
```

## 5. 完整实验流程

```
阶段一: human_cmef_tpa.yaml (ratio=0.3)
  │
  ├── 验证失败 → 检查代码/配置，修改后重跑
  │
  └── 验证通过 ↓

阶段二: mercaptionplus_cmef_tpa.yaml (ratio=0.3)
  │
  ├── 观察 val 指标趋势
  │     ├── val_f1 持续上升 → 可以尝试更大 ratio
  │     └── val_f1 不再上升 → 当前数据量已够
  │
  ├── 逐步提升 ratio (0.3 → 0.5 → 1.0)
  │
  └── 最终选择 test_wheel_f1 最高的配置提交
```

## 6. 模态选择参考

不同 `face_or_frame` 模式使用的输入数据：

| 模式 | 音频 | 视频帧 | 人脸 | 文本 | TPA 使用情况 | 显存 |
|------|------|--------|------|------|-------------|------|
| `audio_text` | 使用 | - | - | 使用 | 仅 audio TPA | ~19 GB |
| `frame_text` | - | 使用 | - | 使用 | 仅 video TPA | ~22 GB |
| `frame` | 使用 | 使用 | - | 使用 | audio + video TPA | ~22 GB |
| `multiframe_audio_frame_text` | 使用 | 使用 | - | 使用 | audio + video TPA + CMEF | ~23 GB |

当前配置使用 `frame` 模式（音频 + 视频帧 + 文本），TPA 在两个模态上均生效。
