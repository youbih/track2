# TPA 有效性验证与帧数优化 —— 对比实验方案

## 1. 问题背景

当前模型使用 TPA（Temporal Pyramid Aggregation）对视频/音频的时序特征做多尺度聚合。TPA 按 scale=[1,2,4] 将时序序列分成不同粒度的组，对每组分别做 cross-attention，再用 inter-scale attention 融合。

**核心问题：原始视频只采样了 8 帧（n_frms=8），TPA 的多尺度设计在这种低帧率下无法发挥作用。**

具体分析：
- scale=1: 8 帧 → 1 组（全局特征，合理）
- scale=2: 4 帧/组 → 2 组（前后半段，勉强有信息）
- scale=4: 2 帧/组 → 4 组（每组仅 2 帧，几乎没有局部时序模式可提取）

这意味着 TPA 最核心的多尺度优势在 8 帧下大打折扣，scale=4 的分组基本没有意义。额外增加的参数（多组 cross-attention、inter-scale attention）可能不仅无益，反而增加了过拟合的风险。

## 2. 对比实验设计

需要跑 4 组实验，验证两个假设：
- **假设 A**：TPA 在 8 帧下不如简单的 mean pooling
- **假设 B**：增加帧数后 TPA 的效果会显著提升

### 实验 1: Mean Pooling + 8 帧（baseline）

| 参数 | 值 |
|------|------|
| video_fusion_type | mean |
| audio_fusion_type | mean |
| n_frms | 8 |
| 其他参数 | 同 stage1 配置 |

目的：建立最简 baseline，验证 TPA 是否有实际收益。

### 实验 2: TPA + 8 帧（当前方案）

| 参数 | 值 |
|------|------|
| video_fusion_type | tpa |
| audio_fusion_type | tpa |
| tpa_scales | [1, 2, 4] |
| n_frms | 8 |
| 其他参数 | 同 stage1 配置 |

目的：当前正在运行的实验，作为对照组。

### 实验 3: TPA + 16 帧

| 参数 | 值 |
|------|------|
| video_fusion_type | tpa |
| audio_fusion_type | tpa |
| tpa_scales | [1, 2, 4] |
| n_frms | 16 |
| 其他参数 | 同 stage1 配置 |

目的：帧数翻倍后，scale=4 每组 4 帧，多尺度分组更加合理。

### 实验 4: TPA + 32 帧

| 参数 | 值 |
|------|------|
| video_fusion_type | tpa |
| audio_fusion_type | tpa |
| tpa_scales | [1, 2, 4] |
| n_frms | 32 |
| 其他参数 | 同 stage1 配置 |

目的：帧数 4 倍后，scale=4 每组 8 帧，多尺度设计充分生效。

## 4. 实施步骤

1. 等当前 Stage 1（实验 2）跑完，记录结果
2. 为实验 1/3/4 分别创建 yaml 配置文件：
   - `train_configs/mercaptionplus_cmef_mean_stage1.yaml`
   - `train_configs/mercaptionplus_cmef_tpa16_stage1.yaml`
   - `train_configs/mercaptionplus_cmef_tpa32_stage1.yaml`
3. 依次运行各组实验的 Stage 1
4. 汇总对比结果

## 5. 评估指标

| 指标 | 说明 |
|------|------|
| val micro_f1 | 主要指标，越高越好 |
| val wheel_f1 | 情绪语义相似度评分 |
| test micro_f1 | 泛化能力参考 |
| 训练时间 | 效率考量 |
| 显存占用 | 硬件可行性 |

## 6. 预期结论与决策

| 实验结果 | 结论 | 后续动作 |
|----------|------|---------|
| 实验1 ≈ 实验2 | TPA 在 8 帧下没用 | 放弃 TPA 或增加帧数 |
| 实验3 > 实验2 | 增加帧数对 TPA 有效 | 使用 16 帧方案 |
| 实验4 > 实验3 > 实验2 | 帧数越多 TPA 越有效 | 使用 32 帧方案（如果显存允许） |
| 实验1 > 所有 TPA | TPA 对本任务无益 | 使用 mean pooling，简单有效 |
| 实验4 显存不足 | 32 帧不可行 | 在 16 帧基础上继续优化 |

最终选择 val micro_f1 最高的方案进入 Stage 2（解冻 LoRA 联合训练）。

## 7. 时间估算

| 实验 | 预估耗时 | 备注 |
|------|---------|------|

| 实验 3 (TPA+16帧) | ~8-10 小时 | CLIP forward 翻倍 |
